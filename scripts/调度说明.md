# 定时任务调度说明

## 两个采集体系并行运行

### 1. 情绪K线采集（已有）

**任务：** `com.kaipanla.sentiment.monitor.plist`
**执行：** 交易日 9:15-15:00，每5分钟
**脚本：** `sentiment_monitor.sh`
**数据：** 情绪分数 → `/tmp/kaipanla/sentiment_data.csv`
**用途：** 情绪K线图

### 2. 开盘啦数据采集（新增）

**任务：** `com.kaipanla.daily.collection.plist`
**执行：** 每日 15:05（盘后全量）
**脚本：** `daily_full_collection.sh`
**数据：** 连板+板块+涨停 → SQLite数据库
**用途：** 策略择时、连板率择时

---

## 调度时间表

| 时间 | 任务 | 说明 |
|------|------|------|
| 9:15-15:00 | sentiment_monitor | 每5分钟采集情绪分数 |
| 15:00 | update_kline_data | 生成情绪日K线 |
| **15:05** | **kaipanla_full_collection** | **开盘啦盘后全量采集** |
| 15:15 | import_data | 导入数据到数据库 |
| 15:30 | daily_report | 生成策略信号报告 |

---

## 关键：两个体系数据互补

```python
# 情绪分数（来自情绪K线体系）
sentiment_score = 32  # 偏冷

# 连板数据（来自开盘啦体系）
lianban_rate = 9.38%  # 冰点

# 综合判断需要两个数据源
if sentiment_score <= 25 and lianban_rate < 10:
    signal = "🧊冰点 + 🧊冰点 = 极寒布局"
```
