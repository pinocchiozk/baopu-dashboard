# Self-Check 对抗性验证清单

*在执行重要任务前使用，防止偷懒和Rationalization（合理化）*

---

## 触发条件

执行以下任务前必须self-check：
1. **投资研究报告** - 决策建议前
2. **策略框架设计** - 新策略上线前
3. **数据引用** - 声明具体数据前
4. **每日报告生成** - 推送前

---

## Self-Check 清单

### 1. 数据来源核查
```
□ 数据来自真实来源（QVeris/财报/权威API）
□ 不是"感觉"、"大概"、"应该"
□ 有工具用了工具吗？
□ 截断了数据吗？（如"只看了前10条"）
```

### 2. 偷懒检测（Rationalization）
```
□ 我有没有走捷径？
□ 有没有把假设当事实？
□ 有没有"应该没问题"的心态？
□ 有没有忽略反面证据？
```

### 3. 完整性检查
```
□ 主要数据都找到了吗？
□ 遗漏了什么？
□ 还有哪些角度没考虑？
□ 我的结论有充分依据吗？
```

### 4. 投资决策专项
```
□ 估值用了真实股价吗？（不是假设的）
□ 数据有时间戳吗？
□ 风险说清楚了吗？
□ 仓位建议合理吗？
```

---

## Rationalization 典型案例（来自Claude Code）

| 偷懒形式 | 例子 | 正确做法 |
|---------|------|----------|
| 假设数据 | "假设股价35元" | 必须用真实股价 |
| 走捷径 | "直接用上次分析" | 重新核实 |
| 忽略异常 | "这个异常值忽略" | 说明原因 |
| 过度自信 | "肯定没问题" | 主动质疑 |

---

## 使用方式

在重要任务前，先过一遍清单：

```
我准备写一份[XX]研究报告
先self-check：
1. 数据来源：□ 真实 □ 可验证
2. 偷懒检测：无走捷径
3. 完整性：主要数据齐全
4. 结论：有充分依据

→ 检查通过后再执行
```
