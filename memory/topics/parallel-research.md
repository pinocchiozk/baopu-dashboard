# 并行投研框架

*投研任务并行分发 + 结果汇聚*

---

## 核心理念

```
串行：问3个问题 → 3 × 10秒 = 30秒
并行：分发3个agent → 1 × 10秒 = 10秒
```

---

## 适用场景

**适合并行的任务：**
- 产业数据调研（查板块 + 查个股 + 查资金）
- 多维度分析（政策面 + 资金面 + 技术面）
- 竞品对比（多家公司同时研究）
- 信息采集（新闻 + 公告 + 研报）

**不适合并行的任务：**
- 有依赖关系的任务（B的结果依赖A）
- 需要逐步深入的探索性研究
- 简单的一次性查询

---

## 并行投研模板

### Phase 1: 任务拆解

当接到投研任务时，先问：
```
这个任务能拆成独立方向吗？
- 方向A：XXX（独立）
- 方向B：XXX（独立）
- 方向C：XXX（独立）
```

### Phase 2: 并行分发

使用 sessions_spawn 并行启动多个subagent：

```python
# 示例：同时研究三个方向
agents = [
    ("产业数据agent", "查一下光模块板块的产业数据..."),
    ("资金流向agent", "查一下主力资金流向..."),
    ("政策面agent", "查一下AI相关政策...")
]

# 并行分发
for name, task in agents:
    spawn(name, task)

# 等待结果汇聚
```

### Phase 3: 结果汇聚

收集各agent结果，汇总成完整报告：

```markdown
## 投研汇总报告

### 产业数据（来自Agent-A）
- ...

### 资金流向（来自Agent-B）
- ...

### 政策面（来自Agent-C）
- ...

### 综合结论
- ...
```

## 投研Agent角色定义

| Agent | 职责 | 输出 |
|-------|------|------|
| 产业研究员 | 板块动态、供需关系、竞争格局 | 产业分析 |
| 资金研究员 | 主力动向、北向资金、融资融券 | 资金分析 |
| 政策研究员 | 宏观政策、行业政策、监管动态 | 政策分析 |
| 技术分析师 | K线形态、均线系统、MACD等 | 技术分析 |
| 基本面分析师 | 财报、估值、业绩预测 | 基本面分析 |

---

## 实施步骤

1. **识别并行机会** - 拿到任务先拆解
2. **定义Agent角色** - 每个agent有明确职责
3. **并行分发** - sessions_spawn with parallel
4. **结果汇聚** - 收集后整合成报告
5. **综合结论** - 主agent给出最终建议

---

## 注意事项

1. **任务必须真正独立** - 不能有依赖关系
2. **结果要有统一格式** - 方便汇聚
3. **超时控制** - 超过时间取已有结果
4. **错误处理** - 某方向失败不影响整体
